# 个人博客/技术社区系统 PRD 与技术规范 (V3.0 旗舰版)

## 1. 项目概述
本项目是一个基于前后端分离架构的现代化个人技术博客与社区系统。系统不仅拥有极具艺术感与互动性的前台体验（如视差插画登录），其底层更是按照**高并发、高可扩展的社区平台架构**进行设计。旨在提供极佳的技术文章阅读与毫秒级检索体验，同时建立完善的创作者生态与社交互动反馈闭环。

---

## 2. 角色与深度核心功能

### 2.1 访客/普通用户 (生态基石)
- **多维文章浏览**：可浏览顶部的“精选推荐”卡片及下方的文章信息流，支持按最新、最热（Hacker News 算法打分）排序。
- **全局沉浸式检索**：通过顶栏搜索框，不仅能搜标题，还能实现**高亮显示的全文检索**和多标签联合筛选。
- **互动与社交（需登录）**：点赞、收藏文章；支持**多级嵌套评论树**（支持 @ 提醒与 Markdown 代码块渲染）；支持关注喜欢的博主。

### 2.2 博主/创作者 (内容引擎)
- **趣味与多渠道登录**：保留“几何插画鼠标视差跟随”页面的同时，扩充 GitHub / 微信扫码一键 OAuth2 登录。
- **专业级内容发布台**：
  - **沉浸式编辑器**：左右分栏的 Markdown 编辑器，支持草稿本地/云端双向自动保存，防止心血丢失。
  - **高阶发布控制**：支持设置封面图、SEO 摘要、多标签绑定（Tags）、**定时发布**，以及历史版本差异对比与回滚。
- **创作者数据中心**：直观展示总阅读量、获赞数、粉丝数增长曲线，以及当前的用户等级（Lv / 徽章外显）。

### 2.3 管理员/系统平台
- **内容全局管控**：拥有独立后台，可对违规文章下架、审核评论、管理分类标签树。
- **WebHooks 触发器**：可配置开放接口，当有新文章发布时，自动推送到钉钉群、企业微信或 Discord 频道。

## 3. UI/UX 视觉与进阶布局规范 (UI/UX Spec)

### 3.1 登录页交互设计 (品牌印记)
- 左侧为几何插画与动态视差效果，右侧为极简毛玻璃登录表单，增加第三方快捷登录的扁平化图标。

### 3.2 登录后主界面布局 (Tech-Community Layout)
主界面严格遵循经典的技术社区网格布局，采用浅灰色背景（如 `#F4F5F5`）突出白色内容卡片。主题色定为**活力橙色 (`#FF8C00`)**。

- **全局沉浸体验**：增加 **Dark/Light 模式一键切换**功能，无缝适配深夜编码阅读场景。
- **顶部导航栏 (Header)**：包含深色顶栏（引入**实时消息红点提醒**铃铛）与浅色分类栏。
- **精选推荐区 (Hero Featured Section)**：4 列横向网格布局 (Grid)，Hover 时卡片产生细腻的上浮与阴影加深动画。
- **主体内容区 (Main Content Area)**：左 70% / 右 30% 的分栏布局。
  - **左侧流**：每篇文章包含置顶标签、大字标题、摘要、**多重彩色 Tags** 以及底部互动元数据。
  - **右侧边栏**：除公告外，增加**“推荐作者榜单”**或**“热门标签云”**组件。

## 4. 旗舰版技术栈选型 (Enterprise Tech Stack)

- **前端架构**: Vue 3 + TypeScript + Vite + Tailwind CSS + **Pinia (状态管理)**。
- **核心后端**: Spring Boot 3.x + Java 21 + Spring Security + OAuth2。
- **持久层**: MyBatis-Plus + MySQL 8.0。
- **检索引擎 (新增)**: **Elasticsearch 8.x**。彻底废弃 MySQL 模糊查询，实现毫秒级分词搜索与相似文章聚合。
- **异步与解耦 (新增)**: **RabbitMQ / RocketMQ**。用于削峰填谷，异步处理高并发的点赞流水入库、以及粉丝的消息通知派发。
- **缓存与性能**: Redis。除了缓存 4 张精选卡片，还用于实现文章阅读量的**防刷防抖 (限流)** 及延迟持久化。
- **对象存储 (新增)**: **MinIO** 或 阿里云 OSS，配合 CDN 加速前端画作与插图加载。

## 5. 核心数据字典设计 (高阶扩展版)

在 V2.0 基础上，为了支撑高并发互动与消息生态，需做如下扩展拆分：

### 5.1 文章扩展表 (`blog_article`)
| 字段 | 类型 | 说明 |
| :--- | :--- | :--- |
| `id` | BIGINT | 主键 |
| `title` ... | ... | 保持 V2.0 基础字段 |
| `status` | TINYINT | **状态 (0:草稿, 1:已发布, 2:定时发布, 3:封禁)** |
| `score` | DOUBLE | **热度分数 (用于 Hacker News 推荐算法)** |

### 5.2 互动流水表 (`blog_interaction_log`) 
*注意：高频点赞绝不能直接更新文章表，必须通过流水表+消息队列异步合并。*
| 字段 | 类型 | 说明 |
| :--- | :--- | :--- |
| `id` | BIGINT | 主键 |
| `user_id` | BIGINT | 发起互动的用户 ID |
| `target_id` | BIGINT | 目标 ID (文章/评论 ID) |
| `action_type`| TINYINT | 动作类型 (1:点赞, 2:收藏, 3:关注) |
| `create_time`| DATETIME | 交互时间 |

### 5.3 消息通知表 (`sys_message`)
| 字段 | 类型 | 说明 |
| :--- | :--- | :--- |
| `id` | BIGINT | 主键 |
| `receiver_id`| BIGINT | 接收者 ID |
| `type` | TINYINT | 消息类型 (1:系统通知, 2:点赞, 3:评论, 4:新粉丝) |
| `content` | JSON | 详细内容 (使用 JSON 灵活存储发起人昵称、文章标题等信息) |
| `is_read` | TINYINT | 是否已读 (0:未读, 1:已读) |

## 6. 开发与演进路线图 (Roadmap)

建议分三期敏捷开发，逐步交付核心价值：

1. **Phase 1 (MVP 基础骨架)**：
   - 跑通 MySQL + MyBatis-Plus，完成文章的增删改查。
   - 重点死磕前端：实现 Tailwind CSS 响应式布局和登录页的“几何插画视差动画”。
2. **Phase 2 (性能与社区化)**：
   - 引入 Redis 缓存首页热点和限流；引入 Elasticsearch 替换原生搜索。
   - 实现文章评论区、点赞功能及前端交互展示。
3. **Phase 3 (生态与高可用)**：
   - 搭建消息中心，引入 RabbitMQ 异步派发红点通知。
   - 实现第三方授权登录 (OAuth2) 与 Webhook 自动推送机制。
